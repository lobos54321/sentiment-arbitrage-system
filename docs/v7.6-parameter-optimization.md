# v7.6 过滤参数优化报告

## 优化背景

基于历史日志数据进行参数网格搜索，寻找最优过滤策略组合。

### 数据样本
- **金狗样本**: 40个 (DeBot 官方金狗/银狗榜单，10x+ 涨幅)
- **噪音样本**: 304个 (被过滤但非金狗的代币)
- **测试组合**: 6400种参数组合

### 优化目标
1. **最大化金狗命中率**: 确保优质代币不被错误过滤
2. **最小化噪音通过率**: 减少无效交易和亏损

---

## 参数搜索空间

| 参数 | 搜索范围 | 最优值 |
|------|----------|--------|
| 七维分阈值 | 15-22 | **17** (原30过高) |
| 聪明钱数量 | 1-5 | **≥2** |
| 趋势上升要求 | true/false | **false** |
| 信号加速要求 | true/false | **true** (关键!) |
| 允许接盘信号 | true/false | **true** |
| 基础分阈值 | 50-65 | **50** |
| 市值上限 | null/50K-2M | **1.5M** |

---

## 最优策略对比

### 策略性能对比

| 策略 | 金狗通过率 | 噪音过滤率 | 精确率 | 综合得分 |
|------|-----------|-----------|--------|----------|
| **BALANCED** (推荐) | 77.5% | 59.9% | 20.3% | 70.45 |
| AGGRESSIVE | 85.0% | 48.0% | 17.7% | 70.21 |
| CONSERVATIVE | 50.0% | 75.7% | 21.3% | 60.26 |
| ULTRA_CONSERVATIVE | 37.5% | 82.2% | 21.7% | 55.39 |

### 综合得分公式
```
综合得分 = 金狗通过率 × 0.6 + 噪音过滤率 × 0.4
```

---

## 关键发现

### 1. 七维分阈值过高
- **原阈值**: 30
- **优化后**: 17
- **影响**: 阈值30时，100%金狗被过滤；阈值17时，77.5%金狗通过

### 2. 信号趋势是关键区分因子
- 要求 `signalTrendType === 'ACCELERATING'`
- 金狗中加速信号占比显著高于噪音

### 3. 聪明钱数量阈值
- ≥2 是最优平衡点
- ≥4 可提高精确率但会漏掉更多金狗

### 4. 市值限制有效
- $1.5M 市值上限是最优平衡点
- 过大的盘子(>$2M)往往已过了最佳入场时机

## 帕累托前沿分析

存在金狗通过率与噪音过滤率的权衡关系：

```
金狗通过 95% → 噪音过滤 14.8% (太宽松)
金狗通过 85% → 噪音过滤 48.0%
金狗通过 77.5% → 噪音过滤 59.9% ← 最优平衡点
金狗通过 65% → 噪音过滤 66.1%
金狗通过 50% → 噪音过滤 75.7% (高精确率)
金狗通过 37.5% → 噪音过滤 82.2% (激进过滤)
```

## 实施方案

### 1. 新增配置文件
```
src/config/filter-params.js
```

包含4种预设策略，可通过修改 `ACTIVE_STRATEGY` 切换：
- `BALANCED`: 推荐，综合最优
- `AGGRESSIVE`: 追求高金狗通过率
- `CONSERVATIVE`: 追求高精确率
- `ULTRA_CONSERVATIVE`: 极致过滤

### 2. 修改 CrossValidator
在 `onNewToken()` 方法中，评分后增加 v7.6 优化过滤检查：
- 位置: 多因子分析之后，决策之前
- 逻辑: 应用 `applyFilter()` 函数检查所有条件

### 3. 启动时打印配置
调用 `printFilterConfig()` 显示当前激活的过滤策略

## 预期效果

使用 BALANCED 策略后：
- **金狗命中**: 从 ~0% 提升到 77.5%
- **噪音过滤**: 保持 59.9% (每10个噪音过滤6个)
- **精确率**: 20.3% (每5个通过的代币中有1个是金狗)
