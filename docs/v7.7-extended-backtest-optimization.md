## 关键发现

### 1. 根本性权衡
**无法同时达到金狗>50%且噪音过滤>50%**

这是数据揭示的根本性约束：
- 高金狗通过率 = 低噪音过滤率
- 高噪音过滤率 = 低金狗通过率

### 2. 最强区分因子: signalTrendType
| 信号类型 | 金狗占比 | 噪音占比 | 区分度 |
|----------|----------|----------|--------|
| ACCELERATING | 29.0% | 9.3% | **3.1x** |
| STABLE | 62.9% | 84.9% | 0.7x |
| DECAYING | 8.1% | 5.8% | 1.4x |

**结论**: 只选 ACCELERATING 信号可以过滤91%噪音，但会漏掉71%金狗。

### 3. 帕累托前沿
存在28个非支配解，代表不同的权衡选择：

```
金狗通过 98.4% → 噪音过滤 2.4%   (几乎无过滤)
金狗通过 90.3% → 噪音过滤 15.2%  ← AGGRESSIVE
金狗通过 85.5% → 噪音过滤 19.3%  ← BALANCED
金狗通过 45.2% → 噪音过滤 45.9%  (最接近平衡点)
金狗通过 29.0% → 噪音过滤 90.9%  ← CONSERVATIVE
金狗通过 1.6%  → 噪音过滤 96.7%  (极端保守)
```

## 策略配置 (v7.7)

### AGGRESSIVE - 金狗优先
```javascript
{
    smCountThreshold: 1,
    baseScoreThreshold: 50,
    requireSignalAccelerating: false,
    // 预期: 金狗90% | 噪音过滤15% | 精确率6%
}
```
**适用场景**: 资金充足，不怕噪音损失，追求不漏金狗

### BALANCED - 综合最优 (默认)
```javascript
{
    smCountThreshold: 1,
    baseScoreThreshold: 50,
    requireSignalAccelerating: false,
    rejectDecayingSignal: true,  // 新增: 排除衰退信号
    // 预期: 金狗85% | 噪音过滤19% | 精确率6%
}
```
**适用场景**: 默认策略，过滤掉明显的衰退信号

### CONSERVATIVE - 精确优先
```javascript
{
    smCountThreshold: 1,
    baseScoreThreshold: 45,
    requireSignalAccelerating: true,  // 关键: 只要加速信号
    // 预期: 金狗29% | 噪音过滤91% | 精确率16%
}
```
**适用场景**: 资金有限，追求每次交易的质量

### ULTRA_CONSERVATIVE - 极致精确
```javascript
{
    smCountThreshold: 2,
    baseScoreThreshold: 45,
    requireSignalAccelerating: true,
    // 预期: 金狗29% | 噪音过滤91% | 精确率16%
}
```
**适用场景**: 极度风险厌恶，宁缺毋滥

| 策略 | 精确率 | 含义 |
|------|--------|------|

| BALANCED | 6% | 每17个信号有1个金狗 |
| CONSERVATIVE | 16% | 每6个信号有1个金狗 |
| ULTRA_CONSERVATIVE | 16% | 每6个信号有1个金狗 |

## 决策建议

### 如果追求覆盖率 (不漏金狗)
使用 **AGGRESSIVE** 策略
- 优点: 90%金狗不会被错过
- 缺点: 大量噪音(85%)会通过，需要其他方式控制损失

### 如果追求精确率 (减少无效交易)
使用 **CONSERVATIVE** 策略
- 优点: 只有9%噪音通过，每个信号质量更高
- 缺点: 会漏掉71%的金狗

### 推荐折中方案
使用 **BALANCED** + 止损控制
1. 过滤层使用 BALANCED (保留85%金狗)
2. 止损层使用严格止损 (控制噪音损失)
3. 仓位层使用动态仓位 (高信号高仓位)
