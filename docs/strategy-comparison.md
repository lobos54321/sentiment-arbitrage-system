# 策略对比：远程策略E vs 新策略 vs 策略C

## 一、远程当前策略（策略E）

### 止损机制
```javascript
// 1. STOP_LOSS: -20%全卖（30s grace period）
if (pnl <= -20 && holdTimeSec >= 30) → 全卖

// 2. FAST_STOP: 60-120秒窗口，从未涨过 & < -15%
if (holdTimeSec >= 60 && holdTimeSec <= 120 && highPnl <= 0 && pnl < -15) → 全卖

// 3. MID_STOP: >60s & < -15% & 从未涨过+10%（连续3s确认）
if (holdTimeSec >= 60 && pnl < -15 && highPnl < 10) → 全卖（3s确认）
```

### 止盈机制（PEAK_EXIT）
```javascript
// 一次性全卖，避免分批滑点叠加
if (highPnl >= 300) → 回撤30%全卖（最低保+100%）
if (highPnl >= 100) → 回撤25%全卖（最低保+50%）
if (highPnl >= 50)  → 回撤20%全卖（最低保+25%）
if (highPnl >= 30)  → 回撤15%全卖（最低保+15%）
```

### 实盘表现（4.75小时，10笔）
- 胜率：40%（4胜6负）
- 总PnL：+7.1%
- 投入：0.80 SOL，收回：0.857 SOL

### 主要问题
1. **MID_STOP误杀严重**：4笔在6-40秒内被杀，全部是价差误触发
   - $ANTS 6秒被杀（-11%），后续+1341%
   - $BILL 38秒被杀（-15%），后续+260%
   - $Belicoin 40秒被杀（-10%），后续+302%

2. **卖早了**：4个赢家都卖在ATH的9-35%位置
   - $4: 卖+28%，后续+308%（只吃到9%）
   - $Cuzco: 卖+27%，后续+266%（只吃到10%）

## 二、策略C（Shadow模式回测）

### 止损机制
```javascript
// 1. STOP_LOSS: -20%全卖（60s grace period）
if (pnl <= -20 && holdTimeSec >= 60) → 全卖

// 2. FAST_STOP: -15%全卖（60s grace period）
if (pnl <= -15 && holdTimeSec >= 60) → 全卖

// 3. 保底机制：TRAIL_STOP最低+10%
if (highPnl > 0 && pnl < max(10, highPnl * 0.65)) → 全卖
```

### 止盈机制
```javascript
// 分批止盈
TP1: +50%卖30%
TP2: +100%卖20%
TP3: +200%卖20%

// MOON_STOP: 涨幅≥200%后，保留55%峰值
if (highPnl >= 200) {
  moonMode = true;
  if (pnl < moonHighPnl * 0.55) → 全卖剩余
}

// TRAIL_STOP: 保留65-70%峰值
if (highPnl >= 300) → 保留70%峰值
if (highPnl >= 100) → 保留75%峰值
if (highPnl >= 50)  → 保留80%峰值
```

### 回测表现（30笔）
- 胜率：100%（30胜0负）
- 总PnL：+12.5%
- 投入：2.36 SOL，收回：2.654 SOL

### 优势
1. **保底机制救了18笔亏损**：策略E的18笔亏损全部转为盈利
2. **Grace period 60秒**：比策略E的30秒更宽松
3. **分批止盈锁利**：TP1/TP2/TP3逐步锁定

### 局限
1. **TP1门槛太高（+50%）**：回测中只有6.7%触发
2. **保底+10%在真实交易中可能降级为+5-8%**（延迟+滑点）
3. **没有解决"卖早了"问题**：在$4上也是+25%就卖了

---

## 三、新策略（目标70%胜率）

### 核心设计：三层防护 + 动态止盈

#### 第一层：Grace Period（60秒保护期）
```javascript
// 买入后60秒内，只触发FAST_STOP（-15%）
if (holdTimeSec < 60) {
  if (pnl <= -15) → FAST_STOP全卖
  else → HOLD
}
```

**vs 策略E**：策略E的30秒太短，MID_STOP在60秒后才生效但仍误杀
**vs 策略C**：相同（60秒）

---

#### 第二层：动态保底机制
```javascript
// 根据持仓时间和峰值动态调整保底线
function getMinPnl(pos) {
  const holdTimeMin = (Date.now() - pos.entryTime) / 60000;

  // 1. 从未盈利（highPnl < 5%）给更多时间
  if (pos.highPnl < 5) {
    if (holdTimeMin < 3) return -15;   // 前3分钟容忍-15%
    if (holdTimeMin < 10) return -12;  // 3-10分钟容忍-12%
    return -10;                        // 10分钟后容忍-10%
  }

  // 2. 曾经盈利，保底线提升
  if (pos.highPnl >= 50) return Math.max(10, pos.highPnl * 0.3);  // 峰值≥50%，保底30%
  if (pos.highPnl >= 30) return Math.max(8, pos.highPnl * 0.25);  // 峰值≥30%，保底25%
  if (pos.highPnl >= 15) return Math.max(5, pos.highPnl * 0.2);   // 峰值≥15%，保底20%
  if (pos.highPnl >= 5) return 3;                                  // 峰值≥5%，保底+3%

  return -10;
}
```

**vs 策略E**：策略E没有保底机制，只有固定止损（-20%/-15%）
**vs 策略C**：策略C是固定保底+10%，新策略是动态保底（根据峰值和时间调整）

**关键差异**：
- 策略C：一旦涨过，保底就是+10%（过于激进，可能过早卖出）
- 新策略：峰值+15%时保底+3%，峰值+50%时保底+15%（更合理）

---

#### 第三层：动态止盈（让利润奔跑）

##### 3.1 分批止盈（降低门槛）
```javascript
TP1: +30%卖20%  // 策略C是+50%
TP2: +60%卖20%  // 策略C是+100%
TP3: +120%卖20% // 策略C是+200%
TP4: +250%卖20% // 新增
```

**vs 策略E**：策略E没有分批止盈，一次性全卖
**vs 策略C**：策略C的TP1门槛+50%太高（只有6.7%触发），新策略降到+30%（预计30%+触发率）

**关键差异**：
- 策略E：一次性全卖，避免滑点叠加（但卖早了）
- 策略C：分批止盈门槛太高，大部分币吃不到
- 新策略：降低门槛，提高触发率，同时保留20%让利润奔跑

---

##### 3.2 MOON_STOP（捕获超级大涨）
```javascript
// 涨幅≥200%进入月球模式
if (pnl >= 200 && !pos.moonMode) {
  pos.moonMode = true;
  pos.moonHighPnl = pnl;
}

// 月球模式：保留55%峰值（允许45%回撤）
if (pos.moonMode) {
  pos.moonHighPnl = Math.max(pos.moonHighPnl, pnl);
  const minPnl = pos.moonHighPnl * 0.55;
  if (pnl < minPnl) → MOON_STOP全卖剩余
}
```

**vs 策略E**：策略E的PEAK_EXIT在峰值+300%时只保留70%（回撤30%），新策略保留55%（回撤45%）
**vs 策略C**：相同（55%峰值）

**关键差异**：
- 策略E：$4涨到+308%，在+215%卖出（70%峰值）
- 新策略：$4涨到+308%，在+169%卖出（55%峰值）
- **新策略给更多上涨空间，但也容忍更大回撤**

---

##### 3.3 时间止盈（防止套牢）
```javascript
// 30分钟后逐步收紧止盈
if (holdTimeMin > 30 && pnl > 10) {
  const tightenFactor = Math.min(0.3, (holdTimeMin - 30) / 10 * 0.05);
  const adjustedRetainRatio = 0.75 - tightenFactor;  // 从75%降到45%

  const minPnl = pos.highPnl * adjustedRetainRatio;
  if (pnl < minPnl) → TIME_EXIT全卖
}

// 60分钟强制退出
if (holdTimeMin > 60) → TIME_EXIT全卖
```

**vs 策略E**：策略E没有时间止盈
**vs 策略C**：策略C没有时间止盈

**关键差异**：新策略独有，防止"涨了又跌回去"

## 六、关键设计差异解析

### 1. Grace Period：为什么新策略是60秒？

**策略E的问题**：
- STOP_LOSS有30秒grace period
- 但MID_STOP在60秒后生效，仍然误杀了4笔（6-40秒内）

**新策略的改进**：
- 前60秒只允许FAST_STOP（-15%）
- 60秒后才启用动态保底机制
- **避免了策略E的MID_STOP误杀**

---

### 2. 保底机制：为什么新策略是动态的？

**策略C的问题**：
- 固定保底+10%过于激进
- 一旦涨到+15%，回撤到+10%就卖出
- 可能错过后续大涨（如$4涨到+308%）

**新策略的改进**：
- 峰值+15%时保底+3%（给更多空间）
- 峰值+50%时保底+15%（锁定利润）
- **平衡了"让利润奔跑"和"锁定利润"**

---

### 3. 分批止盈：为什么新策略降低门槛？

**策略C的问题**：
- TP1门槛+50%，回测中只有6.7%触发
- 大部分币在+20-40%就开始回撤

**新策略的改进**：
- TP1降到+30%，预计30%+触发率
- 4个TP级别，最多锁定80%仓位
- 保留20%让利润奔跑
- **提高了锁利频率，同时保留上涨空间**

---

### 4. MOON_STOP：为什么新策略保留55%峰值？

**策略E的问题**：
- PEAK_EXIT在峰值+300%时保留70%（回撤30%）
- $4涨到+308%，在+215%卖出
- **卖得太早，只吃到70%**

**新策略的改进**：
- MOON_STOP保留55%峰值（回撤45%）
- $4涨到+308%，在+169%卖出
- **虽然比策略E卖得更早，但通过分批止盈已经锁定了部分利润**

---

### 5. 时间止盈：为什么新策略独有？

**策略E和策略C的问题**：
- 没有时间维度的退出机制
- 可能长期持有"涨了又跌回去"的币

**新策略的改进**：
- 30分钟后逐步收紧止盈
- 60分钟强制退出
- **防止套牢，提高资金周转率**

## 八、推荐实施路径

### 方案A：激进（直接上新策略）
- 优势：最大化收益潜力
- 风险：保底机制未经实盘验证
- 适用：愿意承担风险，追求高收益

### 方案B：保守（先上策略C，再迭代到新策略）
- 优势：策略C已有回测验证（100%胜率）
- 风险：TP1门槛太高，可能锁利不足
- 适用：稳健型，先验证保底机制

### 方案C：折中（新策略 + 保守参数）
- Grace Period: 60秒 ✅
- 保底机制: 固定+8%（而非动态）
- 分批止盈: TP1 +35%（而非+30%）
- MOON_STOP: 60%峰值（而非55%）
- 时间止盈: 45/90分钟（而非30/60分钟）

**推荐：方案C（折中）**
- 先用保守参数验证1-2天
- 根据实盘数据逐步调整到目标参数
- 降低风险，同时保留优化空间
