
# 策略说明：Telegram 信号的作用与处理

## 1. Telegram 信号的定位
在目前的 V4.0 架构中，Telegram 信号不再作为“独立入场”的主要依据，而是作为 **“情绪增幅器” (Social Heat Multiplier)**。

*   **DeBot** = 事实层（链上聪明钱、项目基本面、AI 评分）。
*   **Telegram** = 情绪层（社区共识、散户热度、KOL 推推）。

## 2. 它是如何被处理的？
当系统收到 Telegram 信号时，它会进入 `CrossValidator` 的评分模型：

| 维度 | 权重 | 说明 |
| :--- | :--- | :--- |
| **聪明钱行为** | 30% | 来自 DeBot API 的实时买卖数据 |
| **AI 叙事** | 20% | DeBot AI 评分 + Grok LLM 分析 |
| **入场时机** | 15% | 价格是否过高，是否刚发币 |
| **社交热度** | 10% | **Telegram mentions 数 + X 活跃度** 🔥 |
| **安全性** | 10% | 是否丢弃权限、流动性覆盖率 |

### 核心逻辑：
1.  **主动数据补全 (Active Enrichment)**：🔥 **新特性！** 当 TG 信号通过初步的安全审查后，系统会主动调用 DeBot API 获取缺失的 AI 叙事报告和聪明钱分布，不再使用极低的兜底分。
2.  **交叉验证**：如果一个币在 DeBot 被发现是 Gold/Silver 等级，同时在 Telegram 有多个高权重频道在推，它的 **社交热度** 分数会拉满，从而更容易触发 `BUY` 决策。
3.  **深度 LLM 分析**：即使是 TG 信号，通过补全数据后也会进入 LLM 叙事分析，确保逻辑性通过验证。

## 3. 现在的用途
*   **补充深度**：有些项目可能链上数据还没爆发，但 TG 社区已经开始疯传，这能让我们比纯链上工具更早捕捉到“情绪溢价”。
*   **过滤误报**：如果一个币链上看起来不错，但 TG 搜索发现全是负面讨论或零讨论，系统会降低其信任值。

---
> [!TIP]
> **总结**：你不必担心 TG 信号太多，它们大多在后台作为背景数据默默贡献分值。只有通过了 DeBot + TG 双重验证的“高质量信号”才会真正出现在你的买入列表里。
