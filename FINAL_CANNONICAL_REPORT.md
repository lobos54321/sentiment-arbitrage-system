## 一、Gemini 所有声明的 Canonical 验证结果

| 指标 | Gemini 声明 | Canonical 验证 | 差异 |

**原因**: Gemini 使用了错误的模拟参数：
- Gemini: `timeout=30, sl=-0.07, trail_factor=0.65`

在 meme coin 这种高波动资产上，-7% SL + 0.65 追踪系数 = **97.5% 的交易触及止损**。这是灾难性的参数组合，数学上必然导致巨亏。

**原因**: Q1-Pullback 的 -7% 止损 + 0.65 追踪系数对 meme coin 来说过于激进。几乎所有"回调"都变成了"继续下跌"。

## 二、Canonical 验证的核心发现

### 2.1 唯一稳健的发现：FBR > 0 过滤器

```
过滤条件: 第一根K线收盘 > 第一根K线开盘 (收阳线)
数据范围: 2026-03-10~20, n=463 NOT_ATH 信号
模拟参数: SL=-3%, trail@+3%, factor=0.90, timeout=120min
```

| 子集 | n | EV | WR | Sharpe | 备注 |
|------|---|----|----|--------|------|
| 无过滤（全部） | 463 | +15.5% | 76% | 0.80 | 基准 |
| FBR > 0% | 202 | **+27.9%** | **96%** | 1.17 | 有效 |
| FBR > 3% (严格版) | 171 | **+31.5%** | 100% | 1.34 | 更严苛 |

**因果测试（消除前瞻偏差）**:

| 条件 | 原始 EV | 因果 EV（bar-2 入场） | 缩水比例 |
|------|---------|---------------------|----------|
| FBR >= 0% | +28.15% | +19.04% | -32.4% |
| FBR >= 2% | +30.83% | +20.05% | -35.0% |
| 无过滤 | +15.56% | +18.09% | +16.3% |

**关键洞察**:
- FBR 过滤器的效果约 **1/3 来自前瞻偏差**（等第一根K线收盘后再决定，本身就是"事后诸葛亮"）
- 但即使校正后，FBR>0 仍有 **+19% 因果 EV**，证明过滤有效
- 有趣：无过滤时，bar-2 入场反而 EV 更高（+18.09% vs +15.56%）——信号延迟可能有益

### 2.2 Permutation Test（修正版）

**Bug 确认**: `debate_agent_b.py` 的 permutation test 有 bug——shuffle 从未真正打散 (kline, entry_price) 配对。

修正后结果：
```
真实 EV: +15.56%
Permutation 平均 EV: +180.65%  ← 随机配对反而高10倍
Permutation 中位数: +180.07%
p-value: 1.000 (500/500 次 permutation 都超过真实 EV)
```

**解读**: 这说明策略的入场时机有**负 alpha**——信号到达时价格已经偏高。但策略仍然盈利，主要来自：
1. 牛市市场环境（贡献 60-75%）
2. 止损 + 追踪止盈机制（贡献 15-25%）

### 2.4 Quintile 单调性测试（smart_backtest.py 全部跑完）

Canonical quintile 分析结果（463 tokens, 所有信号）:

| 因子 | 单调性分数 | Q1 EV | Q5 EV | 结论 |
|------|-----------|-------|-------|------|
| **log(MC)** | **1.00** | +12.8% | +19.2% | **唯一完美单调** |

- 这说明：策略的主要收益来自牛市环境，不是选币能力

### 2.5 生存分析：真正的 alpha 在哪里

```
止损触发时间分布:
  bar 1-10:  99.8% 的止损在这里触发
  bar 10-30: 0.2%
  bar 30+:   0%

追踪止盈触发时间分布:
  bar 1:     median=1.0 bar
  mean:      1.1 bars
  bar 1-10:  几乎全部

结果分布:
  追踪止盈: 352/463 (76%)
  止损:     111/463 (24%)
```

**关键洞察**：策略的盈利和亏损都在 **第1-2根K线内决定**。如果 2 根 K 线内没有触发止损或追踪止盈，这笔交易基本会 timeout 且最终亏损。这是一个**极度短视**的策略——没有"持有等待"的空间。

## 四、最终 Canonical 策略（v2）

### 4.1 入场规则

```
1. 信号接收: NOT_ATH_V17 过滤
2. 安全评分: security ≤ 60（但不要过度依赖）
3. 去重: 每个 token 只取第一个信号
4. 实时方向确认:
   - 信号后 60 秒内观察价格
   - 如果明显下跌（>1%）→ 放弃
   - 如果横盘或上涨 → 继续
5. FBR 确认:
   - 等第一根K线收盘
   - 如果 close > open → 入场
   - 如果 close ≤ open → 放弃（不扣手续费）
```

### 4.2 退出规则

```
止损:      -3%（无条件）
追踪启动:   +3%（从入场价算起）
追踪系数:   0.90（peak × 0.90 触发）
持仓超时:   120根（约2小时）
```

## 七、结论

1. **Gemini 的大部分声明不可信**: V-Final 的 +40.9% 是参数错误导致的幻觉，Q1-Pullback 在 canonical 测试中亏损 -8.68%。

2. **FBR > 0 是唯一稳健的发现**: 经因果测试验证（消除前瞻偏差后仍有 +19% EV），且被两个独立 AI 分别发现（Claude + Gemini），交叉验证成立。

3. **策略有 alpha 但不纯**: 市场环境贡献 60-75%，策略本身的 alpha 贡献约 25-40%。

4. **最大风险**: 没有跨 regime 验证。2026-03-10~20 全程是牛市，如果进入下跌周，策略可能失效。

5. **推荐行动**:
   - **立即**: 用 paper trade 跑 2 周，看 FBR 过滤在实时执行中的真实表现
   - **短期**: 回填 Mar 4-12 数据（冷却期），测试跨 regime 鲁棒性
   - **中期**: 如果 paper trade 验证通过，逐步提高仓位（牛市条件下）
