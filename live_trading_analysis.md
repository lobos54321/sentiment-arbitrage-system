## ⚠️ 核心问题诊断

### 问题 1：价格数据仍然不准确

**证据：**
- 峰值 +612.7% 但实际只 +14.7%
- 峰值 +893.6% 但实际 +0.0%
- 这些峰值可能是：
  1. 价格单位仍然混乱（USD vs SOL）
  2. Jupiter Price API 返回错误数据
  3. 价格更新延迟导致虚假峰值

**验证方法：**
- 检查最新代码是否部署（USE_PRICE_MONITOR_V2=true）
- 检查 entryPrice 是否使用实际交易价格
- 检查 LivePriceMonitor 是否使用 V2 版本

### 问题 2：PEAK_EXIT 触发太慢

**证据：**
- $PIP: 峰值 +612.7% → 触发时已跌到 +14.7%
- $George: 峰值 +70.9% → 触发时已跌到 -7.1%
- 从峰值到触发，价格暴跌 50-90%

**可能原因：**
1. 价格监控间隔太长（1.5 秒）
2. PEAK_EXIT 回撤阈值太大（30%）
3. 交易执行延迟（Jupiter Quote → Swap → 确认）

### 问题 3：胜率太低（28.4%）

**对比虚拟盘：**
- 虚拟盘胜率：72.0%
- 实盘胜率：28.4%
- 差距：43.6%

**原因：**
- 虚拟盘不考虑执行延迟
- 虚拟盘不考虑滑点
- 虚拟盘价格数据可能更准确

## 🎯 解决方案

### 方案 1：验证价格修复是否生效

3. ✅ 服务是否重启
4. ✅ 日志中是否显示 "使用 V2 版本"

**验证方法：**
```bash
# 检查远程环境变量
# 检查远程日志
# 看是否有 "📡 [LivePriceMonitorV2] 初始化"
```

### 方案 2：优化 PEAK_EXIT 策略

**当前问题：**
- 回撤 30% 才触发，太慢了
- Meme coin 价格波动极快，30% 回撤时已经跌了 50-90%

**建议调整：**
```javascript
// 当前：回撤 30% 触发
if (drawback >= 30) {
  // 卖出
}

// 建议：回撤 10-15% 触发
if (drawback >= 10) {
  // 卖出
}
```

**或者使用时间窗口：**
```javascript
// 峰值后 5 秒内回撤 10% 就卖
if (timeSincePeak > 5 && drawback >= 10) {
  // 卖出
}
```

### 方案 3：降低价格监控间隔

**当前：1.5 秒**
- Meme coin 价格 1.5 秒内可能暴跌 50%

**建议：0.5 秒**
```javascript
this.priceIntervalMs = 500;  // 0.5 秒
```

**风险：**
- API 调用频率增加 3 倍
- 可能触发 rate limit

**缓解：**
- 使用 Jupiter API Key（Ultra 版本）
- 或者只监控持仓 token（不监控所有信号）

### 方案 4：使用 TAKE_PROFIT 策略

**数据支持：**
- 虚拟盘回测：TAKE_PROFIT_30 胜率 72%，期望值 +14.06%
- 实盘 PEAK_EXIT：胜率 28.4%，期望值 -11.2%

**建议：**
```javascript
// 达到 30% 就卖，不等峰值
if (pnl >= 30) {
  // 立即卖出
}
```

**优势：**
- 不依赖价格监控频率
- 不依赖峰值检测准确性
- 稳定获利
